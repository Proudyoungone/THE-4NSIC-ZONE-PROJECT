Drone Forensics Reporting (ECS)

This section documents how Elastic Common Schema (ECS) is used to normalize and analyze drone telemetry, media, and controller activity for DFIR investigations.

âž¤ Purpose

ECS provides a standardized field structure for drone data, allowing flight logs, GPS, media EXIF, network traffic, and operator activity to be correlated into a single forensic timeline.

âž¤ Data Sources Included

  • Flight logs (TXT / CSV / DAT)
  • GPS & GNSS telemetry
  • Remote ID / RF telemetry
  • Photos & videos (EXIF metadata)
  • Controller / mobile app logs
  • Cloud sync / account activity

âž¤ Key ECS Fields Used

Core ECS + custom drone.* namespace:

@timestamp
event.action
event.category
device.model
device.serial_number
user.name
geo.location.lat
geo.location.lon
drone.flight.altitude_msl_m
drone.flight.ground_speed_mps
drone.battery.charge_percent
drone.controller.sticks.pitch

âž¤ Reporting Focus

  • Flight path reconstruction
  • Operator attribution and controller linkage
  • Crash / anomaly analysis (failsafes, geofencing, errors)
  • Correlation with other DFIR evidence (endpoints, network, cloud)
